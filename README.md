# amirsalarsafaei.com

Personal portfolio site — Next.js frontend, Rust (Tonic) backend, PostgreSQL,
gRPC-Web, deployed as Docker images.

Live at **[amirsalarsafaei.com](https://amirsalarsafaei.com)**.

## Features

- 3D world with the exact laptop model as my personal laptop
- Functioning terminal (with `cowsay` 🐮)
- Blog CMS with admin panel
- Spotify "now playing" integration

## Stack

| Layer    | Tech                                                                   |
| -------- | ---------------------------------------------------------------------- |
| Frontend | React 18, TypeScript, Next.js 14, Three.js, gRPC-Web                   |
| Backend  | Rust, Tonic (gRPC), Tokio, Axum, sqlx + PostgreSQL                     |
| Schema   | Protocol Buffers via `buf` → generated TS + Rust crates                |
| Dev      | [devbox](https://www.jetify.com/devbox) (Rust, Node, buf, sqlx-cli, …) |
| Deploy   | Docker / Docker Compose, images published to GHCR                      |

## Development

You need [devbox](https://www.jetify.com/devbox) installed. Everything else
(Rust, Node, Yarn, buf, sqlx-cli, postgres client, openssl, …) is provisioned
by devbox.

```bash
# Enter the dev shell (or use direnv: `direnv allow`)
devbox shell

# Copy env template and fill in secrets
cp .env.example .env

# Install JS deps
(cd frontend && yarn install)
(cd autogenerated && yarn install)

# Regenerate gRPC code from .proto files
just buf-generate

# Run the full stack with hot-reload (postgres + backend + frontend-dev)
just dev

# Or run pieces natively
just backend     # cargo run
just frontend    # yarn dev
```

Run `just` (no args) to list every recipe.

### Database

Migrations live in `backend/migrations/` and are applied automatically by the
`migrate` service in `compose.yaml`. To run them manually against a local DB:

```bash
export DATABASE_URL=postgres://amirsalarsafaeicom:complex-password@localhost:5432/amirsalarsafaeicom
just db-migrate
```

After changing any `sqlx::query!` invocation, refresh the offline cache:

```bash
just db-prepare
```

## Deployment

See **[DEPLOYMENT.md](./DEPLOYMENT.md)**. TL;DR:

```bash
just docker-build
just up
```

CI builds and publishes images to `ghcr.io/<owner>/amirsalarsafaei-com/{backend,frontend}`
on every push to `main` — see [`.github/workflows/build-publish.yml`](./.github/workflows/build-publish.yml).
